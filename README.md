# Fashion Rec Engine (仮)

## 概要 (Overview)
アパレル特化型のレコメンドエンジン（推薦システム）を構築するための、データ収集および分析プロジェクトです。
個人ECサイト（主にBASE等のプラットフォーム）を対象に、各ショップの商品データをスクレイピングし、価格帯やカテゴリ分布を可視化・比較しています。将来的なSaaS化を見据えた技術検証を兼ねています。

## 技術スタック (Tech Stack)
- **言語**: Python 3.12 (仮想環境利用)
- **スクレイピング**: Requests, BeautifulSoup4 (lxml)
- **データ処理**: Pandas
- **可視化**: Matplotlib
- **環境**: VS Code

## 構成ファイル (Files)
- `detective.py`: サイトのHTML構造（クラス名等）を解析・特定するための事前偵察スクリプト
- `combined_base_data.csv`: 取得・統合された商品生データ（現在: 約200件）
- `make_graph.py` / `base_test.ipynb`: データの集計・可視化用スクリプト

## 現在の成果とインサイト (Data Insights)
BASE上のターゲットショップ3社（`pui`, `isxnot`, `amore`）から計200件の商品データを自動取得・統合し、比較分析を実施しました。

**データから見えた事実：**
- **価格帯・戦略の明確な違い**: A社（amore）は平均2,600円台の低単価・多売戦略、B/C社（pui, isxnot）は平均12,000円〜の高単価・ブランド価値戦略をとっており、同じプラットフォーム上でもターゲット層が完全に分かれていることが可視化されました。

## 現在の課題とビジネス価値 (Current Challenge)
現在、商品名に対するキーワードマッチング（例:`シャツ` `pants`）でカテゴリ分類を行っていますが、アパレル特有の自由なネーミング（例：「ランランT」など）により、半数以上の商品が「Other（その他）」に分類されてしまう課題が発生しています。

**👉 考察**
この「カテゴリ分類の難しさ・曖昧さ」こそが、汎用的な検索システムがアパレルECで機能しにくい最大の理由です。逆に言えば、ユーザーの好みや商品の文脈を理解する**「アパレル特化のレコメンドエンジン」には、ここに大きなビジネス価値（介入余地）がある**と考えています。

## 今後の展望 (Next Steps)
- 商品名だけでなく、URL構造や詳細説明文を用いたカテゴリ分類精度の向上（Other率の削減）
- 分析対象ショップを30〜50社へ拡大するための自動巡回クローラーの構築
- 取得したデータに基づく推薦ロジックの基礎検証
